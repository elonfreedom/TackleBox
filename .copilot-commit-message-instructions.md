# Commit message 生成指南（供 Copilot 使用）

请使用下列简单的说明来生成清晰、一致的提交信息，遵循 Conventional Commits 风格：

- 格式：`<type>: <短描述>` 例如：`feat(ui): add settings screen`
- 可选 body：对变更做更详细说明，必要时分行并解释为什么、如何实现、兼容性/迁移步骤。
- 可选 footer：引用 issue 或 BREAKING CHANGE；例如 `Closes #123` 或 `BREAKING CHANGE: ...`。

常用 type：
- `feat`：新增功能
- `fix`：修复 bug
- `docs`：仅文档变更
- `style`：格式（不影响功能的改动）
- `refactor`：重构（既不是 fix 也不是 feat）
- `perf`：性能优化
- `test`：新增或修复测试
- `chore`：构建/工具/依赖等琐事

示例：

1. 简短提交：

```
feat: add sign-in with Apple
```

2. 带 body 的提交：

```
fix: retry on temporary DNS failures

Retry up to 3 times for ephemeral DNS errors to reduce transient failures
when reaching our API. This avoids false negatives reported by the integration tests.

Closes #42
```

3. BREAKING CHANGE 示例：

```
refactor: remove legacy /v1 endpoints

BREAKING CHANGE: clients must migrate to /v2 endpoints; see migration guide.
```

生成建议：提交信息要求使用中文;尽量保持第一行（summary）在 50 字符以内；body 行宽约 72 字符；首字母小写，不要在 summary 末尾加句号。

如果需要项目特定的额外规则，请在此文件顶部添加说明。
